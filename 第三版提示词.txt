基于 Telethon 开发一个 Telegram 文件上传管理系统，要求同时支持 Windows 本地开发/调试 和 Linux Docker 部署。

  直接实现一个完整可运行项目，包含后端、前端、文件监控、上传调度、配置管理、Docker 部署和说明文档。

  核心要求如下：

  1. 实现 Telegram 登录流程。
     支持输入 api_id 和 api_hash。
     支持首次验证码登录。
     支持二次验证密码输入。
     支持会话持久化，服务重启后自动恢复登录状态。
     前端展示登录状态：未登录、等待验证码、等待二次验证、已登录、失败。
  2. 实现多频道管理。
     支持新增、编辑、删除、启用/禁用频道。
     每个频道包含：名称、频道标识、启用状态。
     文件夹配置时，目标频道必须从频道列表下拉选择。
  3. 实现文件夹监控配置。
     支持配置多个上传文件夹，每个文件夹包含：

  - 名称
  - 路径
  - 目标频道
  - 自动上传开关
  - 扫描刷新频率
  - 上传后处理策略
  - 移动目标路径（仅当策略为移动时必填）
    路径设计必须同时兼容 Windows 和 Linux。
    支持多级目录。

  4. 实现自动上传和手动上传。
     自动上传：

  - 可按文件夹独立开启/关闭
  - 定时扫描新增文件
  - 避免重复上传
  - 文件被占用时标记为“占用中”，跳过本轮上传

  手动上传：

  - 支持单选上传
  - 支持多选批量上传
  - 可从文件列表中直接触发上传

  5. 实现上传任务列表和实时进度。
     每个上传任务显示：

  - 文件名
  - 所属文件夹
  - 目标频道
  - 上传状态
  - 进度百分比
  - 错误信息
    状态至少包括：
  - 未上传
  - 上传中
  - 已上传
  - 失败
  - 占用中
    前后端需支持实时或准实时状态刷新。

  6. 实现文件夹浏览页面。
     支持浏览已配置上传目录。
     支持多级目录查看。
     按类型分类展示：

  - 视频
  - 图片
  - 音乐
  - 文档
  - 其他
    前四类支持浏览或预览。
    每个文件展示上传状态。

  7. 实现设置页面。
     设置页集中管理：

  - Telegram API 配置和登录流程
  - 频道配置
  - 监控文件夹配置
  - 自动上传开关
  - 扫描频率
  - 上传后处理策略
    如果选择“移动到另一文件夹”，未设置目标路径时禁止保存。

  8. 实现上传消息规则。
     如果一次上传的文件全部属于媒体文件，优先按 Telegram 媒体组发送。
     如果混有非媒体文件，则使用稳定兼容的普通发送策略。

  caption 生成规则：

  - 按多级目录生成标签
  - 格式：#一级目录 #二级目录 ... 文件名
  - 文件名去掉扩展名
  - 标签字符需要清洗
    示例：
    路径 movies/action/2025/test.mp4
    caption：
    #movies #action #2025 test

  9. 实现上传后处理策略。
     支持：

  - 保留原文件
  - 删除原文件
  - 移动到指定目录
    处理逻辑要有异常保护，避免上传成功后处理失败导致状态错误。

  10. 实现跨平台部署。
     必须提供：

  - Dockerfile
  - docker-compose.yml
  - 配置持久化方案
  - Telethon 会话持久化方案
  - 文件夹挂载方案
    同时保证 Windows 本地可以直接运行和调试。

  11. 技术实现要求。
     优先保证：

  - 上传稳定性
  - 会话恢复可靠
  - 文件状态判断准确
  - 重复上传判定可靠
  - 路径跨平台兼容
  - 结构清晰，便于后续扩展

  12. 交付内容。
     最终输出：

  - 完整可运行源码
  - 前后端实现
  - Telethon 集成逻辑
  - 文件监控与上传调度逻辑
  - Docker 部署文件
  - 本地开发文档
  - 配置说明文档

  实现时优先做可运行版本，不先做过度复杂设计。先保证主流程完整：登录、配置频道、配置文件夹、扫描文件、上传、进度展示、状
  态持久化、Docker 可运行。
